Auxiliary Files

There are a bunch of auxiliary file formats in KM3NeT which are used in different stages of processing and calibration procedures. KM3io.jl supports many of them by defining a container type and extending the Base.read function so that the general pattern is:

f = read("path/to/the.file", FileContainerType)

PMT File

The container type PMTFile is used to load PMT files which are produced by the K40 calibration procedure in Jpp.

Below is an example, using a PMT file from the KM3NeTTestData.jl package.

using KM3io
using KM3NeTTestData

pmtfile = read(datapath("pmt", "calibration_00000117_H_1.0.0_00013757_00013826_1.txt"), PMTFile)
PMTFile containing parameters of 7254 PMTs

Data for individual PMTs can be accessed by indexing using the module ID and the DAQ channel ID of the PMT:

pmtdata = pmtfile[806451572, 4]
pmtdata.gain
0.95

The returned type is PMTData with following fields:

fieldnames(typeof(pmtdata))
(:QE, :gain, :gainSpread, :riseTime_ns, :TTS_ns, :threshold)
